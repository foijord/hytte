# hytte — Kalsneset 27 in 3D

THREE.js reconstruction of the property Kalsneset 27, 4641 Søgne (gnr/bnr 437/109, Kristiansand), built from Kartverket 0.25 m LiDAR and a 16.5 cm Norge i bilder orthophoto. See `data/README.md` for data provenance and coordinate conventions.

## Run

```
python tools/serve.py
```

then open http://localhost:8123/. The server hosts the static files and the save API; three.js is vendored in `js/vendor/`, so no internet is needed.

## Layout

- `index.html`, `js/main.js` — the viewer: terrain mesh from the DTM heightmap with the orthophoto draped, sea plane at NN2000 zero, parcel boundary in yellow, and buildings as editable boxes (orange = on parcel 437/109, gray = neighbours).
- `web/heights.bin`, `web/meta.json` — Float32 heightmap generated from `data/dtm_25cm.tif` by `tools/make_heightmap.py` (default 0.5 m grid; run `python tools/make_heightmap.py 1` for full 0.25 m).
- `web/buildings.json` — oriented building boxes generated by `tools/extract_buildings.py`. Simple footprints get one min-area rectangle; L/Z/T-shaped footprints (single-rect IoU < 0.82) are decomposed into up to 6 boxes by greedy maximal-rectangle covering in the footprint's dominant-angle frame (part ids `<osmid>:<n>`). Base elevation comes from the DTM and height from `DOM − DTM`, sampled inside each box's own footprint area. The raw footprint polygon is kept in each record for a future gabled-roof upgrade.
- `data/extra_structures.json` — structures not in OSM, currently the deck in front of the main cabin, generated by `tools/extract_deck.py`: it flood-fills the LiDAR surface for a locally flat platform at cabin-floor elevation attached to the cabin footprint, and fits cabin-aligned rectangles. Merged into `web/buildings.json` on every `extract_buildings.py` run (type `deck`, drawn wood-brown).
- `data/` — source geodata (DTM, DOM, ortho, parcel polygon, OSM footprints).

Scene coordinates: meters, origin at the address point (E 71428.47, N 6458099.03 in EPSG:25833); x = east, y = height (NN2000), z = south.

## Editing buildings

Click a box to select it, then **t** = move, **r** = rotate (vertical axis only), **s** = scale, **d** = duplicate, **Del** = remove, **Esc** = deselect. The HUD shows the selected building's size, position (EPSG:25833) and heading; every box on the parcel carries a floating dimension label (toggle with the "labels" button).

**Saving:** **Ctrl+S** or the "save" button POSTs the scene to the dev server, which writes `web/buildings_edited.json`. The viewer prefers that file over the generated `web/buildings.json`, so re-running the extractors never clobbers your edits. "reset edits" deletes the edited file and reloads the generated baseline.

**Terrain cut:** the terrain is carved down to each box's base inside its footprint (toggle with the "terrain cut" button), so buildings read as built into the slope — e.g. the storage volume under the deck. Recomputed automatically after every move/scale/delete.

URL parameters for reproducible views: `?cam=x,y,z&tgt=x,y,z&labels=off&cut=off`.

To model the new cabin: delete the three orange cabin boxes and shape replacements (duplicate + scale is quickest), then save.
